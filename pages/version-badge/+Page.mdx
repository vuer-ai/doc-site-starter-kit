The `VersionBadge` and `PackageBadge` components display package name, version, and git hash information in a compact inline badge. They are designed for use within this documentation framework itself.

<div className="p-4 rounded-xl border text-sm" style={{ backgroundColor: 'rgb(var(--color-primary-muted))', borderColor: 'rgb(var(--color-primary) / 0.2)', color: 'rgb(var(--color-text))' }}>
  <strong>Important:</strong> These components are part of the doc-site framework's internal tooling.

  components. The dependency flows one way only: the doc site depends on the library — never the reverse.
  Use this component as a <strong>reference implementation</strong> if you need similar functionality
  in your own project.
</div>

## Usage

There are two components: `PackageBadge` (low-level, explicit props) and `VersionBadge` (high-level, boolean flags that read from build-time constants).

### PackageBadge

Pass explicit values for full control:

<CodeBlock
  language="tsx"
  code={`import { PackageBadge } from '../components/VersionBadge'\n\n<PackageBadge\n  packageName="my-lib"\n  packageFullName="@org/my-lib"\n  versionText="v1.2.3"\n  gitHash="a1b2c3d"\n  linkable\n/>`}
/>

### VersionBadge

Toggle display flags — values are read automatically from build-time constants or `package.json`:

<CodeBlock
  language="tsx"
  code={`import { VersionBadge } from '../components/VersionBadge'\n\n{/* Show package name + version with "v" prefix */}\n<VersionBadge package version prefix />\n\n{/* Version only, no link */}\n<VersionBadge version />\n\n{/* Everything: package, version, git hash, all clickable */}\n<VersionBadge package version prefix hash linkable />`}
/>

## Build-time constants

Configure these in `vite.config.ts`:

<CodeBlock
  language="ts"
  filename="vite.config.ts"
  code={`export default defineConfig({\n  define: {\n    __PACKAGE_VERSION__: JSON.stringify(process.env.npm_package_version),\n    __PACKAGE_NAME__: JSON.stringify(process.env.npm_package_name),\n    __GIT_HASH__: JSON.stringify(process.env.GIT_HASH || 'unknown'),\n  },\n  // ... other config\n})`}
/>
